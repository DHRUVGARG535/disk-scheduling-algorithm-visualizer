Waterfall Model - Project Development

The Waterfall Model is a traditional software development methodology that divides the project into linear phases. Below is how our project, Disk Scheduling Algorithm Visualizer, followed this model.

1. Requirement Analysis

In this phase, we gathered and documented all project requirements. The aim was to create a tool that:

  • Takes disk I/O requests as input
  • Accepts initial head position
  • Allows the user to select an algorithm (FCFS, SSTF, SCAN, LOOK, etc.)
  • Visualizes head movement in a graphical format
  • Displays seek time and access order

2. System Design

We structured the system into multiple components:

  • UI built with HTML, CSS, JavaScript
  • Separate modules for input handling, logic, and output visualization
  • Use of graphs or canvas to show head movements
  • Navigation between pages and responsive design

3. Implementation

Actual coding was done during this phase. The project was implemented using:

  • HTML for structure
  • CSS for design and responsiveness
  • JavaScript for algorithm simulation and logic handling
  • Canvas/SVG for visualizing disk head movements dynamically

4. Testing

We tested the application thoroughly:

  • Verified correct working of each algorithm with various test cases
  • Checked UI responsiveness on different screen sizes
  • Ensured proper input validation (e.g., non-numeric inputs)
  • Bug fixes and optimization for smoother visuals

5. Deployment

After successful testing, we deployed the project:

  • Made project files ready for local and online hosting
  • Shared with mentor and peers for feedback
  • Can be uploaded to GitHub or any static site host like Netlify or Vercel

6. Maintenance

Even after deployment, some tweaks were needed:

  • Minor bug fixing based on user feedback
  • Adding features like reset input or better animation
  • Future scope includes adding more algorithms or disk types
